# 5. PBS — 여러 로봇을 충돌 없이 조율하기

[04장](04_space_time_astar.md)에서 "로봇 한 대의 경로 찾기"를 만들었습니다.
이제 그걸 여러 번 호출해서, 여러 로봇 **전체**가 서로 충돌하지 않는 경로 집합을
만드는 **고수준** 엔진을 봅니다. 이게 **PBS (Priority-Based Search, 우선순위
기반 탐색)** 입니다.

## 5.1 비유: 번호표를 든 손님들

[03장 3.1절](03_reservation_table.md)의 식당 비유를 이어갑니다. 이번엔 "관리자"
역할이 추가됩니다.

> 손님들에게 번호표를 나눠준다. 1번부터 차례로 자기 동선을 짜게 한다.
> 한 사람이 동선을 다 짜면, 그 동선을 장부(예약 테이블)에 적는다.
> 다음 번호 손님은 장부에 이미 적힌 동선을 전부 피해서 자기 동선을 짠다.

**PBS = 이 관리자.** 핵심 규칙은 단 하나입니다.

> **먼저 계획된 로봇의 경로는 절대 바뀌지 않는다. 나중에 계획되는 로봇이
> 그 경로를 피해서 자기 길을 찾는다.**

## 5.2 동작 — agents 벡터 순서가 우선순위 ([02장 2.5절](02_map_and_agent.md) 복습)

```cpp
std::optional<PBSResult> plan(const std::vector<Agent>& agents);
```

```
1. ReservationTable을 깨끗이 비운다.
2. agents[0] (1순위)부터 차례로:
   a. 지금 테이블 상태를 보면서 SpaceTimeAStar로 이 로봇의 경로를 찾는다.
   b. 못 찾으면 → 전체 실패 (std::nullopt 반환).
   c. 찾으면 → 그 경로를 테이블에 등록한다. 이후 모든 낮은 순위 로봇이 이걸 피한다.
3. 전원 성공하면 "로봇id → 경로" 표(PBSResult)를 반환한다.
```

## 5.3 왜 이렇게 하면 충돌이 안 생기는가

i번째 로봇이 경로를 계획할 때, **0번째부터 (i-1)번째까지의 경로는 이미
테이블에 등록되어 있습니다.** [04장](04_space_time_astar.md)의 Space-time A*는
테이블에 점유된 (칸, 시각)을 절대 침범하지 않습니다. 그러므로 i번째 로봇의
경로는 자동으로 0~(i-1)번째 로봇 누구와도 충돌하지 않습니다. 이 논리를
모든 로봇에 대해 순서대로 적용하면, 결과 전체에 충돌이 없다는 게 보장됩니다.

이게 PBS 이름의 "Priority(우선순위)"가 가리키는 것입니다 — 충돌이 생기면
"누가 양보할지"를 그때그때 정하는 게 아니라, **미리 정해놓은 순서**로
모든 충돌을 일괄 해결합니다.

## 5.4 PBSResult — 결과는 "로봇id → 경로" 표

```cpp
using PBSResult = std::unordered_map<int, Path>;
```

```
result[0] = {(0,0), (1,0), (2,0)}   // 로봇 0의 경로
result[1] = {(2,0), (2,1), (2,2)}   // 로봇 1의 경로
```

## 5.5 register_path — 경로를 장부에 옮겨적기 (+ Tail Reservation)

로봇 한 대의 경로가 정해지면, PBS는 그 경로의 **모든 칸**과 **모든 이동**을
테이블에 등록합니다.

```
경로 {(0,0), (1,0), (1,1)}을 등록한다면:
  Vertex: (0,0,t=0), (1,0,t=1), (1,1,t=2)  ← 각 시각의 위치
  Edge:   (0,0)→(1,0) at t=0,  (1,0)→(1,1) at t=1   ← 각 이동
```

Vertex와 Edge 둘 다 등록해야 [01장 1.3절](01_mapf_problem.md)의 두 충돌 종류를
전부 막을 수 있습니다 — 하나만 등록하면 다른 한 종류의 충돌이 새는 구멍이
됩니다.

그런데 이것만으로는 한 가지 문제가 남습니다.

> 로봇 A가 일찍 자기 목적지에 도착해서 거기 가만히 서 있다(작업 중).
> 그런데 A의 경로 배열은 "도착한 시점(t=2)까지"만 기록되어 있다.
> 그 이후 시각(t=3, 4, 5, ...)에 대해서는 테이블에 아무 등록이 없다.
> 나중 순위 로봇이 t=5에 그 칸을 "그냥 지나가는 길"로 쓰려 하면,
> 테이블만 보면 막을 이유가 없어 보인다 — 하지만 실제로는 A가 거기 계속 있다.

그래서 `register_path`는 경로의 칸들을 등록한 뒤, **목적지 칸만 따로
`tail_start`(도착 시각)부터 `max_timestep`까지 전부 추가로 예약**합니다.
이걸 **Tail Reservation**이라 부릅니다 — "도착 후에는 그 자리에 영원히
머문다"고 가정하고 미래의 모든 시각을 미리 채워두는 것입니다. 이 예약은
A의 경로가 등록되는 순간 **이미 끝난 일**입니다 — 나중 순위 로봇이 자기
경로를 찾을 때는 이미 테이블에 있는 정보를 보는 것뿐입니다.

## 5.6 "아직 차례가 안 온 로봇의 목적지"는 보호할 필요가 없다

직관적으로는 이런 걱정이 들 수 있습니다.

> 순위가 B(2번째) → C(3번째)라고 하자. C의 목적지는 (5,5)인데
> C의 차례는 아직 오지 않았다 — 그래서 테이블에는 C의 경로가
> 전혀 등록되어 있지 않다. B가 지금 경로를 찾는 중에 (5,5)를
> 그냥 지나가는 길로 쓰면, 나중에 C가 거기 들어갈 때 문제가
> 생기지 않을까?

결론은 **문제가 생기지 않습니다.** B가 (5,5)를 "지나가는" 것은 특정 시각
(예: t=7)에 머물렀다가 t=8에 떠나는 것이고, 이 사실은 B의 경로가
등록되는 순간 테이블에 (칸,시각) 단위로 정확히 기록됩니다. C의 차례가
오면, C는 [04장](04_space_time_astar.md)의 Space-time A*로 테이블에
점유된 (칸,시각)을 피해서 탐색합니다 — t=7만 피해서 다른 시각에
(5,5)에 들어가면 그만이고, 5.5의 Tail Reservation이 C 자신의 도착
이후를 영원히 보호해줍니다. **충돌은 항상 "같은 칸 + 같은 시각"에서만
생기므로, B와 C가 (5,5)를 서로 다른 시각에 쓰는 것은 처음부터 충돌이
아닙니다.**

이전 버전의 설계는 이 경우를 막기 위해 "모든 로봇의 목적지를 미리
`t=0`부터 `max_timestep`까지 통째로 선점"하는 장치(목적지 임시 선점)를
두었습니다. 그런데 실제로 구현하고 테스트해보니, 이 장치는 다음 두
문제를 일으켰습니다.

1. **불필요하게 보수적입니다.** B가 C의 미래 목적지를 잠깐 지나가는 것
   자체는 충돌이 아닌데, 이 장치는 그것마저 막아버려서 실제로는 풀 수
   있는 입력을 실패로 판정합니다.
2. **다른 로봇의 Tail Reservation을 침범하는 버그로 이어지기 쉽습니다.**
   "내 목적지의 임시 선점만 풀겠다"는 의도로 그 칸의 점유를 지우면,
   하필 그 칸에 이미 다른 로봇이 걸어둔 Tail Reservation(5.5)까지 같이
   지워질 위험이 있습니다 — 두 장치가 같은 (칸,시각) 키 공간을 공유하기
   때문입니다.

그래서 이 설계는 **목적지 임시 선점 장치를 두지 않습니다.** PBS가 보호해야
하는 것은 5.5의 Tail Reservation 하나뿐이며, 그것만으로 모든 vertex/edge
충돌(같은 칸+같은 시각, 같은 이동의 역방향)을 충분히 막을 수 있습니다.

## 5.7 PBS가 모르는 것 — 책임의 경계

PBS는 다음을 **전혀 모릅니다**:

- 장애물이 동적으로 나타날 수 있다는 것 (그건 06장의 일)
- "선택적으로" 일부 로봇만 계획해야 할 수도 있다는 것 (그것도 06장의 일)

PBS가 아는 것은 딱 하나, **"agents 리스트를 순서대로 한 번 끝까지 계획하면
충돌 없는 결과가 나온다"** 뿐입니다. 이렇게 책임을 좁게 유지하기 때문에,
06장에서 "선택적 재계획"이라는 새로운 전략을 PBS *위에* 쌓을 수 있는
것입니다 — PBS 자체를 뜯어고치지 않고도요.

```
[전략]   선택적 재계획         (06장) — "PBS를 언제, 누구에게 다시 돌릴지" 결정
              ↓ 호출
[고수준] PBS  ← 이 장        — "한 번 끝까지 돌리면 충돌 없는 결과"
              ↓ 호출
[저수준] Space-time A*        (04장) — "로봇 한 대의 최단 경로"
```
